## 2. 核心表结构

### 2.1 用户体系

#### `user_information` — 用户主表

| 字段 | 类型 | 说明 |
|------|------|------|
| `UserId` | varchar(40) PK | 邮箱地址 |
| `Name` | varchar(100) | 用户名（注册后加随机 emoji） |
| `Sex` | varchar(10) | 男/女/未知 |
| `Birthday` | date | 生日 |
| `Dept_Type` | int FK | 系别 ID（注册默认 1=未知） |
| `Regdate` | date | 注册日期 |
| `Type` | int FK | 用户类型（1=读者, 3=超管，注册强制为 1） |
| `Can_use` | int | **双语义**：>0 可借数量 / 0=冻结（注册默认 0） |
| `Root` | varchar(64) | 管理编号 |
| `Password` | varchar(64) | MD5+Salt 加密 |
| `Salt` | varchar(64) | 密码盐值 |
| `bio` | varchar(255) | 个人介绍 |
| `login_token` | varchar(100) | 自动登录 Token |
| `token_expire` | datetime | Token 过期时间 |

> **关键变更 v3.0**：`Can_use` 被复用为冻结标记。新用户注册默认 0（冻结），管理员解冻后设为 1。`Dept_Type` 注册默认 1，前端不再显示系别选择。`Type` 强制为 1，禁止自选管理员。

#### `user_behavior_log` — 用户行为日志

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int PK | 自增 |
| `user_id` | varchar(40) | 用户 ID |
| `action_type` | tinyint | 1=click_book, 2=collect_book, 3=read, 5=add_note, 6=ai_summary, 7=book_comment, 9=ai_talk, **34=ai管家**, **35=ai辩论者**, **36=ai出题**, **37=ai深度语音** |
| `book_isbn` | varchar(20) | 关联书籍 |
| `chapter_number` | varchar(64) | 章节号 |
| `content_snapshot` | text | 内容快照 |

#### `user_profile_memory` — AI 用户画像

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_id` | varchar(40) | 用户 ID |
| `date` | date | 画像日期 |
| `reading_preferences` | text | 阅读偏好标签 |
| `emotional_state` | text | 近期情绪状态 |
| `character_sketch` | text | 用户人物刻画（AI System Prompt） |

## 4. 关键字段语义

### 4.1 `Can_use` — 双重语义

| 值 | 含义 | 上下文 |
|----|------|--------|
| 0 | **冻结** | 注册默认值，管理员解冻后改变 |
| 1 | 正常（解冻后） | 管理员操作 toggle-freeze 后 |
| 32 | 可借数量（旧语义） | 历史数据默认值 |

> **代码位置**：`RegisterDTO.toUserInfo()` 设置 `setCan_use(0)`；`AdminServlet.toggle-freeze` 切换 0/1；`BaseServlet.requireLogin()` 检查 `== 0` 拒绝登录。

### 4.2 `status` / `audit_status` — 审核状态

| 表 | 列名 | 审核值 |
|----|------|--------|
| book_comment | `status` | 0=待审核, 1=通过, 2=拒绝 |
| paragraph_comment | `status` | 同上 |
| user_comment | `status` | 同上 |
| book_bottle | `audit_status` | 同上（status 已被漂流状态占用） |

> **管理端**：`AdminServlet` 提供 PUT 审核接口，前端 `AdminPanel.vue` 统一管理。

### 4.3 `Type` — 用户类型

| 值 | 角色 |
|----|------|
| 1 | 普通读者（注册强制） |
| 3 | 超管（需后台分配） |

> 前端 `v-if="currentUserType === 3"` 控制管理员专属功能（编辑章节、有声书生成）。

### 4.4 `status` — 漂流瓶状态

| 值 | 状态 |
|----|------|
| 0 | 漂流中 |
| 1 | 已捞取 |
| 2 | 已过期 |
| 3 | 用户删除 |
