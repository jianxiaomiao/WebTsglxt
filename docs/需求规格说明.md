## 1. 引言

### 1.1 项目背景

随着数字阅读的普及，传统图书馆管理系统缺乏交互性和沉浸感。本项目构建了一个集**图书阅读管理**、**社交互动**、**AI 智能增强**和**有声内容生成**于一体的全栈阅读平台，为用户提供沉浸式、游戏化的阅读体验。

### 1.2 项目目标

| 目标 | 描述 |
|------|------|
| 沉浸式阅读 | 天气氛围、暗黑模式、壁纸引擎、毛玻璃/液态玻璃 UI |
| 阅读管理 | 书架管理、章节阅读、书签笔记、借阅归还、阅读统计 |
| 社交互动 | 实时聊天、好友系统、论坛社区、漂流瓶、共读舱 |
| AI 增强 | AI 宠物管家、AI 辩论者、AI 出题、AI 每日画报、AI 阅读周报、AI 角色分析 |
| 有声内容 | AI 有声书生成（管理员）、TTS 语音朗读、角色配音 |
| 内容治理 | 管理员内容审核（评论/漂流瓶）、用户冻结/解冻、AI 调用限流 |

### 1.3 适用范围

- 高校/机构图书馆的数字化阅读管理
- 面向普通读者的个人阅读社交平台
- 适用于 PC Web 和移动端自适应访问

## 2. 总体描述

### 2.1 系统架构

```
┌────────────────────────────────────────────┐
│              浏览器 (Vue 3 SPA)              │
│  Element Plus + Pinia + Vue Router         │
│  WebSocket | SSE | Axios                   │
├────────────────────────────────────────────┤
│  Vite Proxy  →  /api  /chat/ws            │
├────────────────────────────────────────────┤
│  Tomcat 11  +  Jakarta Servlet             │
│  Servlet 层 (40+ controllers)              │
│  Service 层 (30+ services)                 │
│  DAO 层 (40+ DAO / JDBC + HikariCP)        │
│  Entity 层 (35+ entities)                  │
├────────────────────────────────────────────┤
│  MySQL 9.4  →  library_manager            │
├────────────────────────────────────────────┤
│  火山引擎 ARK API                           │
│  豆包 Chat  +  Doubao-Seedream-4.5 Image   │
└────────────────────────────────────────────┘
```

### 2.2 运行环境

| 层级 | 要求 |
|------|------|
| 前端浏览器 | Chrome / Edge / Safari / Firefox (ES2020+) |
| 后端服务器 | Tomcat 11 + Java 21 |
| 数据库 | MySQL 9.4 |
| AI API | 火山引擎豆包 (Doubao) + Doubao-Seedream-4.5 |

### 2.3 设计约束

- 前后端分离，RESTful JSON API + WebSocket + SSE 通信
- 后端 Servlet 原生架构（`@WebServlet` 注解），非 Spring MVC
- 数据层 JDBC（HikariCP 连接池）
- 前端 SPA，Vite 构建
- WAR 包部署，前端静态文件内嵌

| ID | 需求 |

| NFR-S-01 | 密码 MD5 + Salt 加密，禁止明文 |

## 6. 安全策略

### 6.1 AI 调用限流矩阵

| 端点 | 限流类型 | 限额 | 窗口 |
|------|---------|------|------|
| `PetAiChatServlet`（管家/辩论者/角色分析等） | 每分钟 | 10 次 | 60s |
| `PetAiChatWithMemoryServlet`（AI 对话） | 每日 | 20 次 | 24h |
| `ChatMessageServlet.generateImage`（AI 图片） | 每日 | 5 张 | 24h |
| `AudiobookServlet`（有声书生成） | 管理员 | Type=3 | — |
| 书架 AI 管家 | 前端节流 | 5min/书 | localStorage |

### 6.2 注册安全

```
用户注册 → 邮箱验证码 → 后端强制：
  ├─ DeptType = 1  （未知系别）
  ├─ Type     = 1  （普通读者）
  └─ Can_use  = 0  （默认冻结）
                    ↓
               管理员后台解冻后才能使用
```

### 6.3 内容审核流程

```
用户提交评论/漂流瓶 → status=0 (待审核)
                    ↓
              管理员审核：
              ├─ 通过 → status=1
              └─ 拒绝 → status=2
```
