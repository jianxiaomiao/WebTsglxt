## 1. 设计理念

### 1.1 核心关键词

**沉浸 · 温暖 · 灵动 · 智能**

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **毛玻璃美学** | 全局 Frosted Glass + 可选液态水晶材质 |
| **氛围自适应** | 7 种天气模式 + 暗黑模式（View Transition API 圆形扩散） |
| **宠物陪伴** | 右下角 AI 宠物挂件（PetWidget），气泡对话式交互，5s 自动消失 |
| **动效克制** | 25s 慢速壁纸漫游，书架 3D 翻书飞出动画 |
| **响应式** | PC 全功能 + 移动端折叠 @media max-width: 768px |

## 3. 核心页面设计

### 3.1 登录页 (`/login`)

- 毛玻璃卡片居中
- 邮箱 + 密码输入框
- "记住我"复选框
- 登录 / 注册切换链接
- 暗黑模式自适应

```
┌────────────────────────────┐
│     📝 创建账号             │
│                            │
│  📧 邮箱       [________]  │  ← 邮箱格式校验
│  📨 验证码     [____] [发送]│  ← 60s 倒计时
│  👤 用户名     [________]  │
```

- 分组视图（在读/已读完/想读/自定义分组）

## 4. 管理员后台设计

### 4.1 总览 (`/admin`)

```
┌──────────────────────────────────────────┐
│  📊 内容审核                              │
│  ┌──────┬──────┬──────┬──────┐           │
│  │📚书评 │📝段评 │💬论坛 │🍾瓶子 │           │
│  │  3   │  0   │  5   │  2   │           │
│  └──────┴──────┴──────┴──────┘           │
│                                          │
│  [🟡待审核] [🟢已通过] [🔴已拒绝]         │
│                                          │
│  ┌──────────────────────────────────┐   │
│  │ ID │ 内容 │ 用户 │ 关联 │ 操作    │   │
│  │ 12 │ xxx  │ 1001 │ ISBN │ 通过 拒绝│   │
│  └──────────────────────────────────┘   │
└──────────────────────────────────────────┘
```

### 4.2 审核 Tab 页

**6 个 Tab**：书评审核 | 段评审核 | 论坛评论审核 | 漂流瓶审核 | 用户管理 | 书籍管理

| Tab | 数据源 | 审核 API |
|-----|--------|---------|
| 书评审核 | `GET /admin/book-comments?status=0` | `PUT /admin/book-comment/audit?id=&status=` |
| 段评审核 | `GET /admin/paragraph-comments?status=0` | `PUT /admin/paragraph-comment/audit?id=&status=` |
| 论坛审核 | `GET /admin/user-comments?status=0` | `PUT /admin/user-comment/audit?id=&status=` |
| 漂流瓶审核 | `GET /admin/bottles?auditStatus=0` | `PUT /admin/bottle/audit?id=&status=` |

**审核操作**：点击"通过"→ status=1，"拒绝"→ status=2，操作后刷新列表 + 更新待审计数。

### 4.3 用户管理 Tab

```
搜索框：[________] 🔍

┌─────────────────────────────────────────┐
│ UID │ 昵称 │ 角色组 │ 管控              │
│ 1001│ 张三 │ 读者   │ [封号]           │
│ 1002│ 李四 │ 超管   │ [封号]           │
│ 1003│ 王五 │ 读者   │ [解冻]  ← 已冻结  │
└─────────────────────────────────────────┘
```

- 封号按钮（红色）→ 点击冻结（Can_use=0）
- 解冻按钮（绿色）→ 点击解冻（Can_use=1）
- API：`PUT /admin/user/toggle-freeze?userId=&canUse=`
- 按钮文字随状态切换：`canUse===0 ? '解冻' : '封号'`

### 4.4 书籍管理 Tab

- 搜索（书名/作者/ISBN）
- 分页列表

## 6. 交互规范

### 6.1 权限控制

| 功能 | 可见条件 | 实现 |
|------|---------|------|
| AI 菜单"有声书" | `currentUserType === 3` | `v-if` |
| 编辑/添加章节 | `currentUserType === 3` | `v-if` |
| 管理员后台路由 | `userStore.userType === 3` | 路由守卫 |
| 管理菜单项 | `userStore.userType === 3` | `v-if` ||

### 6.2 错误处理

- 所有 API 调用统一经 `request.js` axios 拦截器 → `code !== 200` 时 `ElMessage.error`
- AI 调用失败 → 捕获后返回友好 JSON 错误，不产生 HTML 500
- 审核操作失败 → 显示 `err.msg` 真实原因
- AI 限流 → 显示还需等待秒数

### 6.3 加载与反馈

- 列表加载：`ElTable` 的 `v-loading` 骨架态
- AI 生成：`ElButton` 的 `:loading` 旋转态（有声书"立即生成"按钮）
- 操作反馈：`ElMessage.success/error/warning` 即时 Toast
- 流式 AI：气泡逐字出现 + 思考中闪烁动画

### 6.4 暗黑模式

- `isDark` 通过 `provide/inject` 全局共享
- 切换使用 View Transition API 圆形扩散动画
- 表单/卡片背景自适应（`.dark-form` class）

### 6.5 响应式断点

| 断点 | 适配 |
|------|------|
| > 768px | 侧边栏展开，全功能布局 |
| ≤ 768px | 侧边栏折叠为汉堡菜单，表单全宽，表格横向滚动 |
